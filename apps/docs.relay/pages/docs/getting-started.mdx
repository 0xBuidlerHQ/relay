# Getting Started

This is a step by step guide to get started using Relay.

## 1: Create `relays.tsx` and declare your types

We recommend having an unique `relays.tsx` files, hosting all of your relays. It will help you organize your code. Create a dedicated module (e.g., `apps/your-app/src/relays.tsx`) that owns the Relay singleton. Begin by defining the TypeScript shapes you care about:

```ts filename="relays.ts" // [!code hl]
type RelayConfig = {};

type RelayStepBase = {
	label: string;
};

type RelayStepTransactionSuccess = {
	type: "transaction";
	txHash: string;
	blockNumber: number;
};

type RelayStepApiCallSuccess = {
	type: "apicall";
};

type RelayStepSuccess = RelayStepTransactionSuccess | RelayStepApiCallSuccess;

type RelayStepError = {
	message: string;
};
```

```ts filename="example.ts" // [!code hl]
import { http, createPublicClient } from 'viem'
import { mainnet } from 'viem/chains'
 
const client = createPublicClient({
  chain: mainnet,
  transport: http(),
})
```

These types keep your configuration options, step metadata, success payloads, and error payloads explicit so the rest of your app consumes typed data. Everything is fully typed.

## 2: Instantiate the Relay and export helpers

With the types in place, call `createRelay` and pass the generics so the returned helpers stay typed across your app. Export the relay for re-use.

```ts
import { createRelay } from "@0xbuidlerhq/relay";

// [!include example.ts] 

const myRelay = createRelay<RelayConfig, StepBase, StepSuccess, StepError>("my-relay");

export { myRelay };
```

This gives you a singleton keyed by `my-relay` with the default config.

## 3: Create the steps

```tsx
import { myRelay } from "../relays";

const steps = [
  createRelayStep({
    id: "id-0",
    label: "This is the first step",
    fn: async () => {
      await new Promise((resolve) => setTimeout(resolve, 1000));
      return StepSuccess({ txHash: "0x", blockNumber: 10 });
    },
  }),
  createRelayStep({
    id: "id-1",
    label: "This is the second step",
    fn: async () => {
      await new Promise((resolve) => setTimeout(resolve, 1000));
      return StepSuccess({ txHash: "0x", blockNumber: 10 });
    },
  }),
];
```

## 4: Initialize the Relay with the steps


```tsx
function PipelineButton() {
  const relay = useRelay();

  return (
    <button
      onClick={() => {
        relay.initialize(steps);
        relay.start();
      }}
      disabled={relay.isRunning}
    >
      {relay.isRunning ? "Processing Web3" : "Run Web3 pipeline"}
    </button>
  );
}
```

Because `steps` use the same metadata and payload types from `relays.tsx`, everything stays strongly typed. The `relay.initialize`/`relay.start` pair resets the flow and begins execution. You can also call `relay.retry()` after failures or `relay.next()` to advance manually, while watching `relay.stepsState` for statuses.

Now your pipeline is wired: `relays.tsx` owns the typed store, the component creates typed steps, and the UI executes them through the Relay helpers.
