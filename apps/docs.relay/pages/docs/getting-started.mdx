# Getting Started

This is a step by step guide to get started using Relay.

## 1: Create `relays.tsx` and declare your types

We recommend having an unique `relays.tsx` files, hosting all of your relays. It will help you organize your code. Begin by defining the TypeScript shapes you care about:

```ts filename="relays_0.ts" // [relays.ts]
type RelayConfig = {};

type RelayStepBase = {
	label: string;
};

type RelayStepTransactionSuccess = {
	type: "transaction";
	txHash: string;
	blockNumber: number;
};

type RelayStepApiCallSuccess = {
	type: "apicall";
};

type RelayStepSuccess = RelayStepTransactionSuccess | RelayStepApiCallSuccess;

type RelayStepError = {
	message: string;
};
```

These types keep your configuration options, step metadata, success payloads, and error payloads explicit so the rest of your app consumes typed data. Everything is fully typed.

## 2: Instantiate the Relay and export helpers

With the types in place, call `createRelay` and pass the generics so the returned helpers stay typed across your app. Last but not least, export the relay for re-use.

```ts filename="relays.ts" // [relays.ts]
import { createRelay } from "@0xbuidlerhq/relay";

// [!include relays_0.ts] 

const myRelay = createRelay<RelayConfig, StepBase, StepSuccess, StepError>("my-relay");

export { myRelay };
```

This gives you a singleton keyed by `my-relay` with your configurations.

## 3: Create the steps

The place where you set your steps doesn't necessarely matter. You can create them in a hook or have them set outside of react. It really is up to you. However, in most setup, the steps will most likely be inside of a react hook.

Here, we are creating 3 placeholders steps. 

```tsx filename="app.tsx" // [app.tsx]
const steps = [
	myRelay.createRelayStep({
		id: "id-0",
		label: "API call",
		fn: async () => {
			await new Promise((resolve) => setTimeout(resolve, 1000));
			return myRelay.StepSuccess({ type: "apicall" });
		},
	}),
	myRelay.createRelayStep({
		id: "id-1",
		label: "Send Tx",
		fn: async () => {
			await new Promise((resolve) => setTimeout(resolve, 1000));
			return myRelay.StepSuccess({
				type: "transaction",
				txHash: "0x",
				blockNumber: 100_000_000,
			});
		},
	}),
	myRelay.createRelayStep({
		id: "id-2",
		label: "API call",
		fn: async () => {
			await new Promise((resolve) => setTimeout(resolve, 1000));
			return myRelay.StepSuccess({ type: "apicall" });
		},
	}),
];
```

Our steps are now created !

## 4: Initialize the Relay with the steps

Initializing the steps put them in memory, ready to fire and execute themselves.

```tsx filename="app.tsx" // [app.tsx]
const App = () => {
	const { initialize } = myRelay.useRelay();

	return (
		<div className="flex gap-4">
			<div>
				<button
					type="button"
					className="bg-cyan-500/20 text-cyan-500 rounded px-2"
					onClick={() => { initialize(steps) }}
				>
					Initialize
				</button>
			</div>
		</div>
	);
};
```

## 5: Start the Relay !

I guess this steps is pretty straightforward !

```tsx filename="app.tsx" // [app.tsx]
const App = () => {
	const { start } = myRelay.useRelay();

	return (
		<div className="flex gap-4">
			<div>
				<button
					type="button"
					className="bg-amber-500/20 text-amber-500 rounded px-2"
					onClick={() => { start() }}
				>
					Start
				</button>
			</div>
		</div>
	);
};
```

## 6: Observe

The last step is to observe what is actually going on inside the relay !

```tsx filename="sidebar.tsx" // [sidebar.tsx]
const Sidebar = () => {
	const { stepsBase, stepsState } = myRelay.useRelay();

	return (
		<div>
			{stepsBase.map((item, index) => {
				const state = stepsState[index];

				return (
					<div className="flex gap-2" key={item.id}>
						<div>{item.label}</div>

						{(() => {
							if (state.status === "success") {
								const payload = state.payload;

								if (payload.type === "apicall") {
									return <div>Success</div>;
								}

								if (payload.type === "transaction") {
									return (
										<div className="flex gap-2">
											<div>{payload.txHash}</div>
											<div>{payload.blockNumber}</div>
										</div>
									);
								}
							}

							if (state.status === "error") {
								return <div>{state.payload.message}</div>;
							}

							return <div>{state.status}</div>;
						})()}
					</div>
				);
			})}
		</div>
	);
};
```

Because `steps` use the same metadata and payload types from `relays.tsx`, everything stays strongly typed. The `relay.initialize`/`relay.start` pair resets the flow and begins execution. You can also call `relay.retry()` after failures or `relay.next()` to advance manually, while watching `relay.stepsState` for statuses.

Now your pipeline is wired: `relays.tsx` owns the typed store, the component creates typed steps, and the UI executes them through the Relay helpers.
