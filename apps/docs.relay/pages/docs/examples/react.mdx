# React example

This snippet mirrors what the Next.js example in `apps/examples.relay/src/app/page.tsx` sets up: a Relay instance keyed by `relay-example` with two steps.

```tsx
import {
  createRelay,
  createRelayStep,
  StepSuccess,
} from "@0xbuidlerhq/relay";

const { useRelay } = createRelay("relay-example");

const steps = [
  createRelayStep({
    id: "load-user",
    fn: async () => {
      const user = await fetch("/api/user").then((r) => r.json());
      return StepSuccess({ user });
    },
  }),
  createRelayStep({
    id: "verify-email",
    fn: async () => {
      // perform verification logic
      return StepSuccess({ verified: true });
    },
  }),
];

function RelayPanel() {
  const relay = useRelay();

  return (
    <section>
      <p>Status: {relay.status}</p>
      <p>Current step: {relay.stepId ?? "idle"}</p>
      <button onClick={() => relay.initialize(steps)} disabled={relay.isRunning}>
        Start flow
      </button>
    </section>
  );
}
```

The docs site and example app both rely on this pattern; you can replace the steps array with your own API calls, blockchain interactions, or persistence logic and let Relay handle the sequencing.
